Command-Line OptionsðŸ”—

SimpleBench provides a variety of command-line options to customize the behavior of your benchmark.

General OptionsðŸ”—

These options control the overall behavior of the SimpleBench framework.

-h, --helpðŸ”—

Show a help message and exit.

--versionðŸ”—

Show SimpleBenchâ€™s version and exit.

--verboseðŸ”—

Enable verbose output for more detailed information during execution.

--quietðŸ”—

Suppress most output. This is useful for automated environments.

--progressðŸ”—

Display a progress bar during benchmark execution.

Reporting OptionsðŸ”—

These options control which benchmarks are run and where results are saved.

--listðŸ”—

List all available benchmarks by group and name, then exit.

--run <benchmark-group, ...>ðŸ”—

Run specified benchmarks selected by group. Multiple groups can be specified by separating them with spaces. You can see a list of available benchmark groups by using the --list option.

Defaults to running all benchmark groups if not specified.

Example of running specific benchmark groups:

Running specific benchmark groupsðŸ”—
  python my_script.py --run group1 group2
--output-path <path>ðŸ”—

Set the output directory for reports saved to the filesystem. Defaults to .benchmarks.

Report TypesðŸ”—

The following sections detail the options for each report type. Most reports can be sent to one or more destinations using space separated values from the following list:

  • console: Print the report to the terminal.

  • filesystem: Save the report to a file in the output path.

  • callback: Pass the report object to a user-defined callback function.

Each report type has an appropriate default destination if none is explicitly specified.

This example report type option generates a rich table report with all result types, printing it to the console and saving it to the filesystem.

Report Type Destination Selection ExampleðŸ”—
  --rich-table console filesystem

Reports can often be filtered to only include specific result types, such as operations-per-second, timing, or memory usage.

This example generates a rich table report with only operations-per-second results, saving it to the filesystem

Report Type Result Filtering ExampleðŸ”—
  --rich-table.ops filesystem

Rich Table ReportsðŸ”—

Generates formatted tables for display in the console. The default destination is console.

--rich-table [{callback,console,filesystem} ...]ðŸ”—

Generate tables for all result types.

--rich-table.ops [{callback,console,filesystem} ...]ðŸ”—

Generate tables only for operations-per-second results.

--rich-table.timing [{callback,console,filesystem} ...]ðŸ”—

Generate tables only for timing results.

--rich-table.memory [{callback,console,filesystem} ...]ðŸ”—

Generate tables only for memory usage results.

CSV ReportsðŸ”—

Exports results to CSV format.

These are easily imported into spreadsheet applications or data analysis tools. The default destination is filesystem.

--csv [{callback,console,filesystem} ...]ðŸ”—

Generate CSV reports for all results.

--csv.ops [{callback,console,filesystem} ...]ðŸ”—

Generate CSV reports only for operations-per-second results.

--csv.timing [{callback,console,filesystem} ...]ðŸ”—

Generate CSV reports only for timing results.

--csv.memory [{callback,console,filesystem} ...]ðŸ”—

Generate CSV reports only for memory usage results.

Graph ReportsðŸ”—

Graph reports generate visual representations of benchmark results, including line graphs and scatter plots. These visualizations can help you better understand statistical characteristics and comparisons across parameterized inputs.

They can only be saved to the filesystem or handled via a callback function.

The default destination for all graph reports is filesystem.

Scatter Plot ReportsðŸ”—

Scatter plot reports display individual benchmark results as points on a graph, allowing for easy comparison of performance across different benchmark parameter values.

--scatter-plot [{callback,filesystem} ...]ðŸ”—

Generate scatter plot graphs for all results.

--scatter-plot.ops [{callback,filesystem} ...]ðŸ”—

Generate scatter plot graphs only for operations-per-second results.

--scatter-plot.timing [{callback,filesystem} ...]ðŸ”—

Generate scatter plot graphs only for timing results.

--scatter-plot.memory [{callback,filesystem} ...]ðŸ”—

Generate scatter plot graphs only for memory usage results.

JSON ReportsðŸ”—

JSON report options allow you to export benchmark results in JSON format, which is useful for programmatic consumption and integration with other tools.

The default destination is filesystem.

--json [{callback,console,filesystem} ...]ðŸ”—

Generate JSON reports only for statistical results.

--json-data [{callback,console,filesystem} ...]ðŸ”—

Generate JSON reports for statistical results and the full raw data.
